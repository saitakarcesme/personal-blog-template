XontentStudio is a local-first content operating system built for managing an X account more seriously.

While ComposeX focused on the writing layer, XontentStudio expands the workflow into a complete studio for generation, review, approval, scheduling, and publishing. It is designed to handle replies, quote posts, reposts, original posts, and thread starters while keeping content varied, readable, and non-duplicative.

The project is intentionally local-first and approval-first. All state lives on the user's machine, the app starts empty instead of relying on fake seeded content, and publishing moves through real review stages instead of pretending automation alone is enough.

### Features
- Local-first architecture with SQLite-based workspace
- Approval-first publishing workflow
- Support for replies, quotes, reposts, original posts, and thread starters
- Draft, approved, scheduled, published, and failed content states
- Feed analysis and source-post import flow
- Planner, queue, analytics, and settings modules
- Real-first setup with no fake seeded content
- Built to run entirely on your own machine
